- A ***docker container*** image, is a lightweight, standalone, executable package of software that includes everything needed to run an application: code, runtime, system tools, libraries, and settings.  

- [Container vs Container Image](https://youtu.be/RqTEHSBrYFw?si=GNLh_UoV01eP-ykc&t=577)
    - A container is a running instance of a container image.
    - A container image is a static file that includes the executable code along with libraries and dependencies required to run an application.
    - When you run a container image, it becomes a container.
